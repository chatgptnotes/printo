Connecting Printo to RealSoft

The ways we can send drawing data into your RealSoft ERP

Prepared for: Coral Business Solutions  |  RealSoft ERP  |  A simple, non-technical guide
 

In one line: what Printo does

Printo reads your approved construction drawings using AI, automatically pulls out the title-block details (drawing number, project name, revision, sheet, scale, and more), checks them for mistakes, and then sends that clean data straight into RealSoft - with no manual typing.

The only open question is how the data travels from Printo into RealSoft. There are three practical ways to do this. This document explains each one in plain language so you can pick the one that suits RealSoft best.

The three ways to connect

Option A - Live API Link RECOMMENDED

What it is: Printo sends each drawing's data to RealSoft instantly over a secure internet link (an "API"), using a private key that RealSoft gives us.

How it feelsA drawing is processed and, seconds later, the record already exists in RealSoft. Fully automatic.
Good becauseReal-time, no files to move by hand, and every send is logged so nothing gets lost.
Things to noteRealSoft needs to expose a web address (endpoint) that accepts the data.
What we needA web address (base URL) and a security key from RealSoft / Coral.
StatusAlready built in Printo - we only need the address and key to switch it on.
Option B - Excel / CSV File Import SIMPLE FALLBACK

What it is: Printo creates a RealSoft-ready spreadsheet (Excel or CSV) of the drawing data. That file is then loaded into RealSoft through RealData Hub - either by a person or on a schedule.

How it feelsLike handing RealSoft a perfectly formatted spreadsheet, ready to import in one click.
Good becauseVery simple, no live connection required, and it uses RealData Hub which already exists.
Things to noteIt works in batches, so it is not instant. The spreadsheet must match RealSoft's import template exactly.
What we needThe Excel / CSV import template (the exact column names) from RealSoft / Coral.
StatusQuick to add once we have the template.
Option C - Read-Back Check OPTIONAL / LATER

What it is: Printo can also read records back from RealSoft - for example, to confirm a drawing was saved, or to avoid creating the same record twice.

How it feelsA safety net that double-checks RealSoft after sending data.
Good becausePrevents duplicates and gives extra confidence the data landed correctly.
Things to noteThis is a "nice to have" for later, not needed to go live.
What we needRead access to RealSoft's Data Portal.
StatusOptional - can be added after the main connection works.
One method we will NOT use: automatic "push" alerts from RealSoft (webhooks / event callbacks). RealSoft does not offer these, so our design does not depend on them. No action needed from your side.

Side-by-side comparison

Method Speed Setup effort Best for Status in Printo
A. Live API Link Instant (real-time) Low - needs URL + key Day-to-day automatic flow Built - awaiting key
B. Excel / CSV Import Batch (not instant) Low - needs template Simple start / offline fallback Quick to add
C. Read-Back Check On demand Optional Duplicate prevention later Optional add-on

Our recommendation

Start with Option A (the Live API Link). It is real-time, fully automatic, and the code is already built in Printo - we only need a web address and a security key from your side. We suggest keeping Option B (Excel / CSV) ready as a simple fallback, and adding Option C (Read-Back) later for extra safety.

What we need from RealSoft / Coral to go live

[ ]RealSoft web address (base URL) for the test environment.
[ ]Security key (API key / token) so Printo can connect safely.
[ ]Exact "import" address (the endpoint that accepts new drawing records).
[ ]The exact field names RealSoft expects (the DrawingMaster columns), so the data matches perfectly.
[ ]The Excel / CSV import template - only if you prefer Option B.
[ ]5-10 sample approved drawings so we can test accuracy end-to-end.

Is it secure?

Yes. Every connection uses a private security key (a "Bearer token") that only RealSoft and Printo know. Until RealSoft provides the real key, Printo runs in a safe simulation mode - it prepares the exact data it would send, but does not send anything live. This lets us test everything safely before connecting to your real system.
