# ERP Realsoft - Setup on a New Device

How to get this project running after a fresh `git clone` / `git pull`. The code is in git;
**secrets and dependencies are not** — you supply those locally. Follow top to bottom.

Project: ERP Realsoft / SABI RFQ to BOQ pipeline. Next.js 14 (App Router), Supabase, Claude via Nexaproc AI gateway, and Vercel.

---

## 1. Prerequisites

| Tool | Version | Check |
|---|---|---|
| Node.js | 20+ (22/24 fine) | `node --version` |
| npm | bundled with Node | `npm --version` |
| Git | any recent | `git --version` |
| Supabase CLI | via `npx` (no install) | `npx supabase --version` |
| Vercel CLI | via `npx` (optional) | `npx vercel --version` |

---

## 2. Clone the repo

```bash
git clone <new-realsoft-repo-url>
cd <realsoft-repo-folder>
```

(If you already have it: `git pull origin main`.)

---

## 3. Install dependencies

```bash
npm ci            # exact versions from package-lock.json (preferred)
# or: npm install
```

`node_modules/` is gitignored, so this step is always required on a new device.

---

## 4. Create the environment file (the critical step)

The real `.env` is **not in git** (it holds secrets). You must create it locally.
`.env.example` is the checklist of every key the app needs.

**Easiest — pull the real values from Vercel** (they already live there):

```bash
npx vercel login
npx vercel link          # select the ERP Realsoft project
npx vercel env pull .env.local
```

**Or — copy manually**: get the filled-in `.env` from the original device (USB, password
manager, or secure share — never commit it) and place it at the repo root. Use
`.env.example` to confirm nothing is missing.

### What the env file must contain (see `.env.example` for the full list)
- **Supabase:** `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`, `SUPABASE_SERVICE_ROLE_KEY`
- **Auth:** `JWT_SECRET`
- **AI gateway:** `USE_AI_GATEWAY`, `NEXAPROC_GATEWAY_URL`, `DRAWTOBOQ_AIAS_KEY` (compatibility key name; and/or `ANTHROPIC_API_KEY` for the legacy direct path)
- **Uploads:** no email credentials are required; users create projects and upload drawings directly.

Verify they're all present:

```bash
npm run check-env
```

## 5. Run it

```bash
npm run build     # runs prebuild (generates PDFKit fonts) + next build
npm run dev       # http://localhost:3001
```

For a production-style local run: `npm run start` (after `npm run build`).

| Command | Purpose |
|---|---|
| `npm run dev` | Dev server on http://localhost:3001 |
| `npm run build` | Production build (prebuild fonts + `next build`) |
| `npm run start` | Run the production build locally |
| `npm run lint` | ESLint |
| `npm run check-env` | Verify required env vars are present |

## 6. Database migrations (only if you change the schema)

The live DB schema is managed by the **Supabase CLI**, not by hand-pasting SQL.
Full runbook: [`supabase/MIGRATIONS.md`](supabase/MIGRATIONS.md).

The CLI link is per-device (it's gitignored), so authenticate on each machine:

```bash
npx supabase login                                    # paste personal access token
npx supabase link --project-ref osxsaqiobvpmeqckpjqp  # prompts for DB password
```

Day-to-day change:

```bash
npx supabase migration new my_change   # creates a timestamped file
# edit the file in supabase/migrations/
npx supabase db push                   # applies to the live DB
git add -A && git commit -m "db: my change" && git push
```

## 7. What is NOT in git (you must provide it per device)

| Item | Why | How to get it |
|---|---|---|
| `.env` / `.env.local` | secrets | `vercel env pull` or secure copy |
| `node_modules/` | large/derived | `npm ci` |
| `.next/` | build output | regenerated by `npm run build` |
| `.vercel/` | per-device link | `npx vercel link` |
| `supabase/.temp/` | per-device CLI link | `npx supabase login` + `link` |
| PDFKit fonts | generated | created by `prebuild` during `npm run build` |

## 8. Deployment

- Push to `main` → Vercel auto-builds and deploys **Production**.
- Push any other branch / open a PR → Vercel **Preview** URL.
- CI (`.github/workflows/ci.yml`) runs lint + typecheck on every PR/push.
- Full deploy reference, env var table, and rollback steps: [`deploy.md`](deploy.md).

## 9. Common gotchas

- **App boots but every API call fails** → `.env` missing or incomplete. Run `npm run check-env`.
- **`build` fails on `prebuild`** → `node_modules/pdfkit` missing; re-run `npm ci`.
- **`permission denied for table` (42501)** → service_role grants not applied; see
  `supabase/migrations/20260616_grant_users_table.sql` and the runbook.
- **Supabase 500s** → `SUPABASE_SERVICE_ROLE_KEY` (server) and `NEXT_PUBLIC_SUPABASE_ANON_KEY`
  (client) set and not swapped.
- **Only `NEXT_PUBLIC_*` env vars reach the browser** — anything else is server-only.
